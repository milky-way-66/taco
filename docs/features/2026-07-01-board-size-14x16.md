# Board sizes 14×14 and 16×16

**Date:** 2026-07-01  
**Status:** Implemented  
**Related:** `TacXO/Models/GameSettings.swift`, `README.md`

## Summary

Add two larger board sizes (14×14 and 16×16) to Settings so players can play on bigger grids with existing win-length rules.

## Acceptance criteria

1. Settings board picker lists `14×14` and `16×16` alongside existing sizes.
2. A game on 14×14 accepts plays at (0,0) through (13,13) and rejects out-of-bounds cells.
3. A game on 16×16 accepts plays at (0,0) through (15,15) and rejects out-of-bounds cells.
4. Boards that do not fit on screen use scrollable layout (same behavior as 10×10 on small devices).
5. AI uses heuristic move selection on boards larger than 10×10 (no minimax) to stay responsive.

## Behavior

### In scope

- New `BoardSize` enum cases with correct `dimension` values
- Picker populated via `BoardSize.allCases`
- README updated with new sizes

### Out of scope

- New win-length defaults per board size
- AI difficulty tuning specific to 14×14 / 16×16
