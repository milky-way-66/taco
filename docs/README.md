| `docs/features/` | Small, focused change specs |

## Workflow (short)

1. Write or update a spec (acceptance criteria required)
2. Write failing tests from those criteria
3. Implement minimal code to pass
4. Update docs and mark spec `Implemented`

See [development-workflow.md](development-workflow.md) for full rules.
